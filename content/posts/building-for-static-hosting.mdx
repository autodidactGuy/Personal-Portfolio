Static hosting works best when the content model is explicit.

Instead of spreading text across React components, the portfolio now treats content as a first-class asset:

- JSON powers structured UI sections such as hero blocks, stats, and navigation.
- MDX powers rich long-form entries such as blog posts, projects, and case studies.
- Validation runs at build time so missing fields fail early instead of quietly shipping broken pages.

## Why this matters

For a portfolio, a static-first approach keeps deployment simple while still giving content authors a proper editing surface through Decap CMS.

```ts
export async function getStaticProps() {
  const posts = await getPosts();

  return {
    props: {
      posts,
    },
  };
}
```

That keeps the site Git-based, reviewable, and easy to rebuild through GitHub Actions.
